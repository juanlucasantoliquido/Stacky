# Project Rules (STACKY-AGENTS)

- Preferir cambios pequeños y verificables; no refactorizar fuera del scope pedido.
- Ejecutar tests relevantes antes de cerrar cambios.
- No usar comandos destructivos sin confirmacion explicita.
- Priorizar respuestas breves con pasos accionables.
- Este es un agente framework en TypeScript/Node; respetar los contratos de interfaz existentes.
- Antes de leer multiples archivos, usar el repo map para elegir solo los relevantes.
- No generar mas de 2-3 archivos por turno para no superar la ventana de contexto.
