0. RESUMEN RÁPIDO

🎯 Qué debe desarrollar el Developer

Agregar en la barra superior de Resumen de Actividad un nuevo contador: Promesas a Vencer en 7 días, contando promesas vigentes (unidad: promesa, no lote) de lotes asignados al usuario. Implementar en el flujo existente de resumen sin crear pantalla nueva.

🧪 Cómo probar (paso a paso)

  1. Ingresar con usuario Pacifico y abrir home de Agenda.
  2. Verificar que se visualiza el nuevo contador en la barra resumen.
  3. Validar que el valor coincide con la query SQL de control para ese usuario.
  4. Ingresar con usuario no Pacifico y verificar que el contador no aparece.
  5. Refrescar página y verificar recálculo del contador.

1. TRADUCCIÓN FUNCIONAL → TÉCNICA

Requerimiento funcional (resumen)

Mostrar en pantalla principal un indicador con promesas vigentes cuyo vencimiento esté entre hoy y hoy+6 días, acotado a cartera del usuario y visible solo para instancia Pacifico.

Solución técnica propuesta

Extender el dataset de resumen de agenda con una tabla adicional de conteo (PROMESASVENCER7) y renderizar ese valor en la barra del control de menú superior, con validación de visibilidad por mandante Pacifico.

Flujo actual (hoy)

  1. MasterForm renderiza AISMenu.
  2. AISMenu.LlenarDatosAgendaPersonal() llama a RSFac.Agenda.Resumen().
  3. RSFac delega a RSBus.Agenda.Resumen().
  4. RSBus delega a RSDalc.Agenda.Resumen()/ResumenPool().
  5. Se devuelven 4 tablas: pendientes hoy, pendientes otros días, realizadas hoy, lotes asignados.

Flujo propuesto (después)

  1. (sin cambio) Se obtiene resumen como hoy.
  2. [CAMBIO] En DAL agregar tabla PROMESASVENCER7 con COUNT de promesas vigentes en ventana [hoy, hoy+6].
  3. [CAMBIO] En AISMenu leer esa tabla y renderizar nuevo bloque.
  4. [CAMBIO] Mostrar bloque solo para usuarios asociados a mandante Pacifico (RMANDANTE.MACOD='0010').

Queries de soporte (solo SELECT)

-- Q1: Conteo esperado por usuario en ventana 7 días
SELECT COUNT(*)
FROM RCPAGO cp
JOIN RLOTE lo ON lo.LOCOD = cp.CPLOTE
JOIN RUSUPERF up ON up.UPID = lo.LOPERFIL AND up.UPFECBAJA IS NULL
WHERE up.UPUSUARIO = @Usuario
  AND cp.CPHISTORICO = '0'
  AND cp.CPFECVEN BETWEEN CONVERT(varchar(8), GETDATE(), 112)
                      AND CONVERT(varchar(8), DATEADD(day, 6, GETDATE()), 112)
  AND ISNULL(cp.CPCANPAG,0) < ISNULL(cp.CPCANCOM,0);

-- Q2: Usuario pertenece a Pacifico
SELECT up.UPUSUARIO, up.UPID
FROM RUSUPERF up
JOIN RPERFEMP pe ON pe.PEIDPERF = up.UPID
WHERE up.UPUSUARIO = @Usuario
  AND up.UPFECBAJA IS NULL
  AND pe.PEEMPRESA = '0010';
