0. RESUMEN RÁPIDO

🎯 Qué debe desarrollar el Developer

Agregar dos campos de solo lectura (Corredor Principal y Riesgo de Cliente) en la sección superior de FrmDetalleClie.aspx, visibles únicamente cuando Const.EsInstanciaPacifico == true. El Corredor Principal se deriva con una query TOP 1 sobre ROBLG (por mayor DEMORATOT, desempate OGFECMOR ASC). El Riesgo de Cliente ya viene en el DataSet CLIENTE desde GetCliente() (campo CLRIESGOENT).

🧪 Cómo probar (paso a paso)

  1. Verificar que Web.config tiene <add key="EsInstanciaPacifico" value="1" />.
  2. Abrir la agenda y navegar a un cliente con al menos una obligación: SELECT TOP 5 LOCOD FROM RLOTE ORDER BY LOCOD;
  3. Entrar al Detalle de Cliente (FrmDetalleClie) y verificar que aparecen los campos Corredor Principal y Riesgo de Cliente en la sección superior.
  4. Verificar que Corredor Principal muestra el OGCORREDOR de la obligación con mayor DEMORATOT del lote.
  5. Verificar que Riesgo de Cliente muestra el valor de RCLIE.CLRIESGOENT para ese cliente.
  6. Verificar que si se cambia EsInstanciaPacifico a 0 en Web.config y se reinicia la app, los campos desaparecen.

1. TRADUCCIÓN FUNCIONAL → TÉCNICA

Requerimiento funcional (resumen)

Mostrar en la sección "Datos de Identificación e Información General del Deudor" de FrmDetalleClie dos nuevos campos de solo lectura: (1) Corredor Principal derivado de la obligación con mayor importe de deuda vigente del lote; (2) Riesgo de Cliente leído directamente del registro del deudor. Ambos visibles solo en instancia Pacífico.

Solución técnica propuesta

Corredor Principal: Nuevo método GetCorredorPrincipal(string codCliente) en la capa DAL → BUS → FAC sobre la tabla ROBLG con JOIN a RDEUDA. Ordena por DEMORATOT DESC y OGFECMOR ASC (empate = mora más antigua). Devuelve CORREDOR_PRINCIPAL como DataSet fusionado en dsGeneral.

Riesgo de Cliente: El campo CLRIESGOENT YA está en el SELECT de RSDalc.Cliente.GetCliente() (línea 23 del método) y por lo tanto ya disponible en dsGeneral.Tables["CLIENTE"]. Solo requiere binding en CargoBloqueCliente().

Flag de instancia Pacífico: Mecanismo nuevo a introducir: Const.EsInstanciaPacifico (bool estático) leído de AppSettings["EsInstanciaPacifico"] en Global.asax.cs. Controla la visibilidad de ambos controles.

Flujo actual del sistema (cómo funciona HOY)

  1. Usuario navega a FrmDetalleClie (lote en Session["lote"]).
  2. CargoDatosPagina() llama a facCliente.GetCliente(CodCliente) → retorna tabla CLIENTE con CLRIESGOENT ya incluido.
  3. CargoBloqueCliente(DataSet) lee CLIENTE y LOTE y llena los abf* del header (ApellidoNombre, Documento, NumCliente, NivelMora, CategoriaCliente, TipoCliente, AtrasoTotal, Usuario, Perfil).
  4. No existen controles para Corredor Principal ni Riesgo de Cliente.

Flujo propuesto (cómo debe funcionar DESPUÉS)

  1. (sin cambio) Usuario navega a FrmDetalleClie.
  2. [CAMBIO] CargoDatosPagina() además llama a facOblig.GetCorredorPrincipal(CodCliente) y fusiona en dsGeneral.
  3. (sin cambio) facCliente.GetCliente() → CLRIESGOENT ya incluido en tabla CLIENTE.
  4. [CAMBIO] CargoBloqueCliente() lee y asigna abfCorredorPrincipal.Value y abfRiesgoCliente.Value.
  5. [CAMBIO] Visibilidad controlada por Const.EsInstanciaPacifico (setter en InicializoControles() o en el ASPX con Visible="false" + activación en code-behind).

Queries para obtener datos de prueba

-- P01: Lote con múltiples obligaciones con corredor — validar selección por max DEMORATOT
SELECT TOP 10 OGLOTE, OGCORREDOR, DEMORATOT, OGFECMOR
FROM ROBLG INNER JOIN RDEUDA ON DEOBLIG = OGCOD
WHERE OGCORREDOR IS NOT NULL AND OGCORREDOR <> ''
ORDER BY OGLOTE, DEMORATOT DESC;

-- P02: Lote sin corredor en ninguna obligación — validar campo vacío
SELECT TOP 5 OGLOTE FROM ROBLG
WHERE OGLOTE NOT IN (SELECT DISTINCT OGLOTE FROM ROBLG WHERE OGCORREDOR IS NOT NULL AND OGCORREDOR <> '')
AND OGLOTE IS NOT NULL;

-- P03: Cliente con CLRIESGOENT no nulo
SELECT TOP 5 CLCOD, CLRIESGOENT, CLRIESGOSIS FROM RCLIE
WHERE CLRIESGOENT IS NOT NULL AND CLRIESGOENT <> '';

-- P04: Empate en DEMORATOT — validar desempate por OGFECMOR ASC (mora más antigua)
SELECT OGLOTE, OGCORREDOR, DEMORATOT, OGFECMOR
FROM ROBLG INNER JOIN RDEUDA ON DEOBLIG = OGCOD
WHERE OGCORREDOR IS NOT NULL AND OGCORREDOR <> ''
AND OGLOTE IN (
  SELECT OGLOTE FROM ROBLG INNER JOIN RDEUDA ON DEOBLIG = OGCOD
  WHERE OGCORREDOR IS NOT NULL
  GROUP BY OGLOTE HAVING COUNT(DISTINCT DEMORATOT) = 1 AND COUNT(*) > 1
)
ORDER BY OGLOTE, OGFECMOR ASC;

5. NOTAS PARA EL DESARROLLADOR

Convenciones del proyecto a respetar

  • Dual SQL obligatorio: el método GetCorredorPrincipal() debe manejar SQL Server (TOP 1) y Oracle (WHERE ROWNUM = 1 con subquery) igual que GetObligaciones() en el mismo archivo.
  • Sanitización obligatoria: el parámetro codCliente debe pasarse con cFormat.StToBD(codCliente), nunca concatenado directamente.
  • Patrón de delegación: DAL → BUS → FAC, siguiendo exactamente el patrón de GetHistoricoObligaciones() en cada capa.
  • El facOblig ya está declarado en FrmDetalleClie.aspx.cs (línea 16): RSFac.Obligaciones facOblig = new RSFac.Obligaciones();. No crear nueva instancia.

⚠️ Aclaración sobre CLRIESGOENT vs CLRIESGOSIS

La BD tiene dos campos de riesgo en RCLIE:

  • CLRIESGOENT = "Riesgo Entidad" → riesgo que Pacífico asigna al cliente.
  • CLRIESGOSIS = "Riesgo Sist. Finan." → riesgo calculado por el sistema financiero externo.

Recomendación técnica: Usar CLRIESGOENT como "Riesgo de Cliente" (es el riesgo interno de Pacífico). Si el analista funcional confirma que el campo es el externo, cambiar a CLRIESGOSIS. Ambos ya están en el SELECT de GetCliente(); solo requiere cambiar el nombre del campo en el binding de CargoBloqueCliente. Confirmar con analista funcional antes de desplegar.

Precauciones

  • No modificar la query de GetCliente() — CLRIESGOENT ya está en el SELECT, no hay que agregar nada.
  • La tabla "CORREDOR_PRINCIPAL" en dsGeneral se agrega solo si Const.EsInstanciaPacifico == true. En CargoBloqueCliente(), usar Ds.Tables.Contains("CORREDOR_PRINCIPAL") antes de acceder para evitar KeyNotFoundException.
  • La visibilidad de los controles debe setearse en InicializoControles() (ya existe en FrmDetalleClie.aspx.cs, línea ~150) para que se active en PostBack también.
  • Const.EsInstanciaPacifico es un campo estático de clase → es global a toda la aplicación. No usar instancias de Const.

Patrón de referencia en el proyecto

Para el flag de instancia: ver cómo Comun.Const.Aplicacion = "1"; se setea en Global.asax.cs línea 30 y cómo se usa en RSDalc/Idioma.cs línea 138 (if (Const.Aplicacion == "1")).

Para el método nuevo en DAL/BUS/FAC: ver GetHistoricoObligaciones() en los tres archivos de la capa Obligaciones — es el patrón a replicar exactamente.

Query de derivación del Corredor Principal (SQL Server)

-- GetCorredorPrincipal — SQL Server
SELECT TOP 1 ISNULL(OGCORREDOR, '') AS CORREDOR_PRINCIPAL
FROM ROBLG
INNER JOIN RDEUDA ON DEOBLIG = OGCOD
WHERE OGLOTE = {cFormat.StToBD(codCliente)}
  AND OGCORREDOR IS NOT NULL
  AND OGCORREDOR <> ''
ORDER BY DEMORATOT DESC, OGFECMOR ASC;

-- Equivalente Oracle
SELECT CORREDOR_PRINCIPAL FROM (
  SELECT NVL(OGCORREDOR, '') AS CORREDOR_PRINCIPAL
  FROM ROBLG
  INNER JOIN RDEUDA ON DEOBLIG = OGCOD
  WHERE OGLOTE = {cFormat.StToBD(codCliente)}
    AND OGCORREDOR IS NOT NULL
    AND OGCORREDOR <> ''
  ORDER BY DEMORATOT DESC, OGFECMOR ASC
) WHERE ROWNUM = 1;
