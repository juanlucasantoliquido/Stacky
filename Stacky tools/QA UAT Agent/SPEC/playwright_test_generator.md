# SPEC — `playwright_test_generator.py`

## 1. Propósito

Genera un archivo `.spec.ts` por escenario UAT, listo para ejecutarse con Playwright. La generación es **completamente determinista** — aplica plantillas Jinja2 con los datos del `ScenarioSpec` y el UI map; ningún LLM interviene. Es imposible que el generator produzca un selector no presente en el UI map.

## 2. Alcance

**Hace:**
- Por cada `ScenarioSpec` en el input, genera un `<scenario_id>.spec.ts` con estructura fija
- Traduce cada paso del `ScenarioSpec` al método Playwright correspondiente usando el UI map
- Traduce cada oráculo a una assertion `expect(...).<matcher>` con mensaje de error custom
- Inyecta hooks de evidencia (screenshots antes/después de cada paso)
- Incluye login en `beforeEach` y cleanup en `afterEach`
- Si un selector del `ScenarioSpec` no está en el UI map → marca el escenario como `BLOCKED SELECTOR_NOT_FOUND` (no genera el archivo)

**NO hace:**
- Invoca Playwright, ADO ni LLM
- Valida que el selector funcione en el browser real
- Ejecuta los tests

## 3. Inputs

### CLI

```bash
python playwright_test_generator.py \
  --scenarios evidence/70/scenarios.json \
  --ui-maps cache/ui_maps/ \
  --out evidence/70/tests/ \
  [--template templates/playwright_test.spec.ts.j2] \
  [--verbose]
```

| Arg | Tipo | Descripción |
|---|---|---|
| `--scenarios <path>` | str | JSON de `uat_scenario_compiler.py` |
| `--ui-maps <dir>` | str | Carpeta con `<pantalla>.json` de `ui_map_builder.py` |
| `--out <dir>` | str | Carpeta destino para los `.spec.ts` generados |
| `--template <path>` | str | Plantilla Jinja2; default: `templates/playwright_test.spec.ts.j2` |
| `--verbose` | flag | Logs detallados a stderr |

### Env vars (inyectadas en el `.spec.ts` como process.env)

| Var | Descripción |
|---|---|
| `AGENDA_WEB_BASE_URL` | URL base del Agenda Web |
| `AGENDA_WEB_USER` | Usuario de login |
| `AGENDA_WEB_PASS` | Password de login |

Los tests generados leen estas vars via `process.env.<VAR>`; nunca están hardcodeadas.

### Estructura del `.spec.ts` generado

```typescript
// ADO-70 P01 — Búsqueda SIN filtros activos muestra todos los lotes
// Generated by playwright_test_generator.py v1.0.0

import { test, expect } from '@playwright/test';

const BASE_URL = process.env.AGENDA_WEB_BASE_URL!;
const USER = process.env.AGENDA_WEB_USER!;
const PASS = process.env.AGENDA_WEB_PASS!;

test.describe('ADO-70 | P01 — Búsqueda SIN filtros activos', () => {

  test.beforeEach(async ({ page }) => {
    // SETUP: login
    await page.goto(`${BASE_URL}Login.aspx`);
    await page.fill('#txtUsuario', USER);
    await page.fill('#txtPassword', PASS);
    await page.click('#btnIngresar');
    await page.waitForLoadState('networkidle');
  });

  test('P01 busqueda sin filtros muestra todos los lotes', async ({ page }) => {
    // DATA: usuario=PABLO, empresa=0001

    // SETUP
    await page.goto(`${BASE_URL}FrmAgenda.aspx`);
    await page.waitForLoadState('networkidle');
    await page.screenshot({ path: 'evidence/70/P01/step_00_setup.png' });

    // ACTION
    // [STEP 01] click btn_buscar
    console.log('[STEP 01] click btn_buscar');
    await page.click('input[value=\'Buscar\']');
    await page.waitForLoadState('networkidle');
    await page.screenshot({ path: 'evidence/70/P01/step_01_after.png' });

    // ASSERTIONS
    await expect(page.locator('.gvAgendaAut tr'), 'P01: grid debe tener más de 0 filas').toHaveCount({ min: 1 });
    await expect(page.locator('#divMensaje'), 'P01: mensaje de lista vacía no debe ser visible').toBeHidden();

    // CLEANUP
    await page.context().clearCookies();
  });

  test.afterEach(async ({ page }) => {
    await page.context().clearCookies();
  });
});
```

## 5. Contrato de uso

**Precondiciones:**
- El JSON de `scenarios` valida contra `scenario_spec.schema.json`
- Para cada pantalla del `ScenarioSpec`, existe `<pantalla>.json` en `--ui-maps`

**Postcondiciones:**
- Por cada escenario `status: generated`, existe el `.spec.ts` en `--out`
- Por cada escenario `status: blocked`, NO se genera archivo; la razón está en `results[].reason`
- Ningún `.spec.ts` generado contiene un selector no presente en el UI map
- Todos los `.spec.ts` usan credenciales via `process.env`, nunca hardcodeadas

**Idempotencia:** sí — regenerar con el mismo input sobreescribe los mismos archivos

## 6. Validaciones internas

- Por cada `target` en `pasos[]` y `oraculos[]`, verifica que `alias_semantic` o `asp_id` existan en el UI map de la pantalla → si no existe → escenario a `BLOCKED SELECTOR_NOT_FOUND`
- El `.spec.ts` generado debe compilar con `tsc --noEmit` antes de reportarse como generado exitosamente
- Si la plantilla Jinja2 falla el render → error `template_render_failed`
- Credenciales hardcodeadas en el template → error de build (lint del template antes de render)

### Mapeo de acciones del ScenarioSpec a métodos Playwright

| Acción `ScenarioSpec` | Método Playwright |
|---|---|
| `navigate` | `page.goto(BASE_URL + target)` |
| `click` | `page.click(selector)` |
| `fill` | `page.fill(selector, valor)` |
| `select` | `page.selectOption(selector, valor)` |
| `wait_networkidle` | `page.waitForLoadState('networkidle')` |
| `wait_visible` | `expect(locator).toBeVisible()` |

### Mapeo de oráculos a assertions Playwright

| Oracle `tipo` | Matcher Playwright |
|---|---|
| `equals` | `toHaveText(valor)` |
| `contains_literal` | `toContainText(valor)` |
| `contains_semantic` | `toContainText(valor)` (evaluación semántica postprocesada por `assertion_evaluator`) |
| `count_gt` | `toHaveCount({min: N+1})` |
| `count_eq` | `toHaveCount(N)` |
| `visible` | `toBeVisible()` |
| `invisible` | `toBeHidden()` |
| `state` | `toHaveAttribute('disabled', '')` etc. |

## 7. Errores esperados

| Código | Cuándo |
|---|---|
| `invalid_scenarios_json` | Input no parseable o no valida contra schema |
| `ui_map_not_found` | No existe `cache/ui_maps/<pantalla>.json` para la pantalla del escenario |
| `template_render_failed` | Error de Jinja2 al renderizar el template |
| `tsc_compile_failed` | El `.spec.ts` generado no compila (indica error en el template) |

## 8. Dependencias

- `jinja2` — para renderizar la plantilla
- `tsc` (TypeScript compiler) — para validar el `.spec.ts` generado
- Python 3.8+ stdlib
- Sin LLM

## 9. Ejemplos de uso

```bash

python playwright_test_generator.py \
  --scenarios evidence/70/scenarios.json \
  --ui-maps cache/ui_maps/ \
  --out evidence/70/tests/

  --scenarios evidence/70/scenarios.json \

  --out evidence/70/tests/ \

```

## 10. Criterios de aceptación

- [ ] Para los 6 escenarios ejecutables del ticket 70: genera 6 `.spec.ts` con `status: generated`
- [ ] Cada `.spec.ts` compila con `tsc --noEmit` sin errores
- [ ] Ningún `.spec.ts` contiene credenciales hardcodeadas
- [ ] Un escenario cuyo `target` no está en el UI map produce `status: blocked, reason: SELECTOR_NOT_FOUND`
- [ ] Todos los archivos generados tienen los bloques `// DATA`, `// SETUP`, `// ACTION`, `// ASSERTIONS`, `// CLEANUP`
- [ ] El JSON retornado valida contra el schema de output del generator
