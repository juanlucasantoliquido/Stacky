# SPEC — `selector_discovery.py`

## 1. Propósito

Helper interno de `ui_map_builder.py`. Dado un elemento del DOM (con sus atributos: kind, role, label, asp_id, data-testid, texto), decide el **selector Playwright más robusto** posible y provee fallbacks ordenados por robustez. Garantiza que solo se usen selectores en un orden de preferencia determinista, sin LLM.

## 2. Alcance

**Hace:**
- Aplica una jerarquía de selectores en orden de preferencia y elige el mejor disponible
- Devuelve un nivel de robustez (`high`, `medium`, `low`)
- Provee fallbacks para casos donde el selector principal no sea estable

**NO hace:**
- Invoca Playwright — trabaja solo con los metadatos del elemento
- Accede a la red ni a ADO
- Sugiere alias semánticos — eso lo hace `ui_map_builder` con LLM

## 3. Inputs

### Función Python (módulo, no CLI)

```python
from selector_discovery import discover_selector

result = discover_selector({
    "kind": "select",
    "role": "combobox",
    "label": "Empresa",
    "asp_id": "ddlEmpresa",
    "data_testid": None,
    "text": None
})
```

| Campo | Tipo | Descripción |
|---|---|---|
| `kind` | str | `input`, `select`, `button`, `table`, `div`, etc. |
| `role` | str | Role ARIA del elemento |
| `label` | str\|None | Texto del label asociado |
| `asp_id` | str\|None | ID del elemento en el DOM (puede ser generado por ASP.NET) |
| `data_testid` | str\|None | Valor del atributo `data-testid` si existe |
| `text` | str\|None | Texto visible del elemento (para botones/links) |

## 4. Outputs

### Dict Python

```python
{
    "selector": "#ddlEmpresa",
    "robustness": "high",   # high | medium | low
    "fallbacks": [
        "getByLabel('Empresa')",
        "[id$='ddlEmpresa']"
    ],
    "warning": None   # o "requires_data_testid_from_dev" si robustness==low
}
```

## 5. Contrato de uso

### Jerarquía de selectores (en orden de preferencia)

1. **`getByTestId('<data_testid>')`** — si `data_testid` está presente → `robustness: high`
2. **`getByRole('<role>', {name: '<label>'})`** — si role + label son únicos → `robustness: high`
3. **`getByLabel('<label>')`** — si label es único → `robustness: high`
4. **`#<asp_id>`** — si el ID es estable (no contiene `_ctlNN_` ni patrones autogenerados de ASP.NET) → `robustness: medium`
5. **`text='<texto>'`** — para botones/links con texto único → `robustness: medium`
6. **CSS posicional / XPath calculado** — último recurso → `robustness: low` + `warning: requires_data_testid_from_dev`

**Idempotencia:** sí — función pura, mismo input → mismo output.

## 6. Validaciones internas

- Un ID de ASP.NET autogenerado se detecta por regex: contiene `_ctl\d+_`, `__`, o tiene longitud > 60 chars con prefijos como `ctl00$`
- Si `robustness == low`, siempre se agrega `warning: "requires_data_testid_from_dev"`
- Un selector con `data_testid` vacío string `""` se trata como `None`

## 7. Errores esperados

Esta función es pura y no lanza excepciones. Si el input está vacío o malformado, retorna el selector más conservador posible con `robustness: low`.

## 8. Dependencias

- Python 3.8+ stdlib únicamente — sin dependencias externas
- Sin LLM

## 9. Ejemplos de uso

```python
from selector_discovery import discover_selector

# Elemento con data-testid → selector ideal
result = discover_selector({"data_testid": "filtro-empresa", ...})
# → {"selector": "getByTestId('filtro-empresa')", "robustness": "high", ...}

# Select ASP.NET con ID estable
result = discover_selector({"asp_id": "ddlEmpresa", "label": "Empresa", "role": "combobox", ...})
# → {"selector": "#ddlEmpresa", "robustness": "high" (via getByLabel), ...}

# ID autogenerado → low
result = discover_selector({"asp_id": "ctl00_ContentPlaceHolder1_ddlEmpresa", ...})
# → {"selector": "...", "robustness": "low", "warning": "requires_data_testid_from_dev"}
```

## 10. Criterios de aceptación

- [ ] `ddlEmpresa` (sin `_ctlNN_`) con label `"Empresa"` → `robustness ≥ medium`
- [ ] `ctl00_ContentPlaceHolder1_txtFecha` → `robustness: low` con warning
- [ ] `data_testid` presente → `robustness: high` y selector `getByTestId(...)`
- [ ] Función pura: mismos inputs siempre producen mismo output
- [ ] Nunca lanza excepción, incluso con input malformado
